Introduction

The Quill & Radio is an application software (or so-called 'graphical user interface (GUI)') created by me in Python, specifically using tkinter. It is a standalone application with which I use to quickly record information I just heard from a radio show. The output of the application is saved in a text file (.txt).

As a licensed amateur radio operator, while using a notepad or spreadsheet, I found that information is not as organized as I expected. So, I decided to create my own application that can do the things I wanted.

Procedure

Below is the first version of the application. Left contained station information and digital clock. Right contained entry boxes where a user would enter information and the application would display the inputs on the left. Visually, it looks barren.

Below is the second version. I added Matplotlib - a module that can create maps. But embedding Matplotlib slowed down the application's execution time, and it raised an error with .after(), which was the digital clock. The size of the map was clunky. To solve this issue, I trashed the old design and coded the UI again from scratch.

The final version is shown below. The challenge was building a logic that doesn't raise errors. For example, the map should not update until a user selects the the cities (the "Listening to" box is disabled for this reason).

Anyhow, the application works fine. I learned so many things from doing this side project. I will apply these lessons when I build a new application.
